# User Requirements

## Main Idea:

- **Car Website Project (Comprehensive Automotive Platform)**  
  A website dedicated to cars that provides **detailed and well-structured information** about different models.

## Functional Requirements

### Main

- `FR-1` **Authentication System**  
   Implement user authentication, including:
  - Sign up (registration)
  - Login
  - Logout

- `FR-2` **Car Search Functionality**  
   Allow users to search for car models efficiently using filters or keywords.

- `FR-3` **Car Preview Pages**
  each car has a dedicated page fot it

- `FR-4` **Car Price Management**  
   Provide functionality to **update and manage car prices dynamically**.

### Extra:

- `FR-5` **3D Overview of each car**

- `FR-6` **Cars are categorized**

- `FR-7` **CRUD perations on car's data**

- `FR-8` **Wishlists**

## Non-functiona Requirements

- `NFR-1` **Project Scope & Pages**  
  The project must include **at least 8 pages**, with multiple modules and features. Teams are expected to extend functionality beyond the basics to demonstrate creativity.

- `NFR-2` **Project Quality**  
  The project must be **realistic, complete, and advanced**, not a simple demo.

- `NFR-3` **Themes & Layout Variations**  
  Implement **at least two themes** (e.g., light and dark):

- Each theme must have its own **color scheme and layout**
- Layout includes positioning and structure of elements
- Theme selection must be **saved and restored** (persistent)

- `NFR-4` **Layouts Requirement**  
  The website must include **at least 8 distinct, complex layouts**.

- `NFR-5` **Semantic Structure & Content Formatting**  
  Use proper:

- Headings, paragraphs, and formatting
- HTML entities
- Semantic HTML elements (`<header>`, `<article>`, `<footer>`, etc.)

- `NFR-6` **CSS Usage & Layout Control**

- Use **external CSS stylesheets** (and internal if needed)
- Demonstrate strong layout control
- Apply CSS transformations, translations, and transitions

- `NFR-7` **Links & Media**

- Links with CSS customization
- Image links with captions
- Images and image maps (when applicable)
- Icons
- Videos and audio

- `NFR-8` **Tables**

- Tables with CSS customization
- Advanced tables
- Nested elements inside tables

- `NFR-9` **Forms (Functional)**  
  Include fully functional forms such as:

- Login forms
- User registration/profile forms
- Complaints/feedback forms

- `NFR-10` **Search Functionality**  
  Implement search with paginated results and multiple layouts:

- List view
- Grid view
- Detailed view

- `NFR-11` **IFrames**  
  Include iframe usage where applicable.

- `NFR-12` **Dynamic Behavior (JavaScript / jQuery)**  
  Implement dynamic UI logic, such as:

- Show/hide elements
- Add/remove/change elements or attributes

- `NFR-13` **Custom Form Validation (JavaScript)**  
  Validate forms using JavaScript or jQuery only:

- Do not rely on HTML5 built-in validation
- Display custom validation messages

- `NFR-14` **Browser Storage Usage**  
  Use JavaScript to read from and write to:

- LocalStorage or SessionStorage

- `NFR-15` **Advanced UI Components**  
  Include advanced elements such as:

- Tables, iframes, image maps, icons, and multimedia
