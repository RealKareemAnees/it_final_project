# System Requirements

This document defines the system-level requirements for the Car Website Project and maps each requirement to the approved functional and non-functional requirements.

## System Scope

- The system must include a frontend website, backend server, and database.
- The website must contain at least 8 pages and at least 8 distinct layouts.
- The system must support public users, registered users, and administrators.

## Detailed System Requirements

### `SR-1` Landing Page (Home)

The landing page must introduce the platform and provide fast access to core features.

- Include introductory content describing the website purpose and available services.
- Include a hero section with image links and captions.
- Include a featured cars gallery.
- Include links with custom CSS styling.
- Include icons, images, and at least one multimedia block (video and/or audio).
- Include an optional image map when suitable for navigation.
- Include a search bar that forwards user input to the search page.

Traceability: `FR-2`, `NFR-1`, `NFR-7`, `NFR-10`

### `SR-2` Global Header and Navigation

The system must provide a reusable semantic header visible on all main pages.

- Use semantic HTML structure for the header/navigation area.
- Provide a global navigation bar with links to all primary pages.
- Support responsive behavior for desktop and mobile screens.

Traceability: `NFR-1`, `NFR-5`, `NFR-6`

### `SR-3` Global Footer

The system must provide a reusable semantic footer visible on all main pages.

- Include copyright/project information.
- Include quick links (About, Contact, Search, Login, Admin where applicable).
- Use semantic HTML and consistent styling.

Traceability: `NFR-1`, `NFR-5`

### `SR-4` About Page

The system must provide a dedicated About page.

- Describe project goals and platform capabilities.
- Present team or project information in a well-structured layout.
- Use semantic content sections and rich formatting.

Traceability: `NFR-1`, `NFR-5`

### `SR-5` Contact and Feedback Page

The system must provide a dedicated Contact page with a functional feedback form.

- Display contact information.
- Provide an email/contact form for messages or complaints.
- Implement custom JavaScript validation and custom error messages.
- Do not use browser-native HTML5 validation as the only validation mechanism.

Traceability: `NFR-1`, `NFR-9`, `NFR-13`

### `SR-6` Authentication Module (Login/Signup/Logout)

The system must support full user authentication.

- Provide login and signup interfaces (popup or page-based).
- Validate authentication forms using custom JavaScript logic.
- Implement session handling and logout functionality.
- Protect restricted pages/actions for authenticated users only.

Traceability: `FR-1`, `NFR-9`, `NFR-13`

### `SR-7` Theme Management

The system must support at least two visual themes.

- Provide at least light and dark themes.
- Each theme must include a distinct color scheme and layout behavior.
- Provide a theme switch control accessible from the UI.
- Persist user theme preference using localStorage or sessionStorage.

Traceability: `NFR-3`, `NFR-14`

### `SR-8` Search and Results Page

The system must provide advanced car search and categorized filtering.

- Support keyword search.
- Support category/filter-based search.
- Display paginated results.
- Provide multiple result layouts: list view, grid view, and detailed view.

Traceability: `FR-2`, `FR-6`, `NFR-10`

### `SR-9` Single Car Details Page

The system must provide a dedicated details page for each car.

- Display car specifications and structured information.
- Display a car image gallery.
- Include 3D preview support for the car (embed/iframe/model viewer as applicable).
- Present pricing and key comparison details clearly.

Traceability: `FR-3`, `FR-5`, `NFR-2`, `NFR-11`, `NFR-15`

### `SR-10` Backend Server and Data Layer

The system must provide backend services and persistent data storage.

- Provide API endpoints for authentication and user management.
- Provide API endpoints for car search, retrieval, and category filtering.
- Store user accounts, car data, categories, and pricing in a database.
- Enforce authorization rules for admin-only actions.

Traceability: `FR-1`, `FR-2`, `FR-4`, `FR-6`, `FR-7`

### `SR-11` Admin Panel

The system must provide an admin interface for full car data management.

- Implement Create, Read, Update, and Delete operations for car records.
- Allow dynamic price updates.
- Support category assignment and update.
- Present data in advanced table-based views with CSS customization.

Traceability: `FR-4`, `FR-7`, `NFR-8`, `NFR-15`

### `SR-12` Wishlist

users should be able to have wishlists

- wishlist pages
- add/remove to/from wishlist

Traceability: `FR-8`

## Cross-Cutting System Constraints

- Use external CSS stylesheets as the main styling method.
- Use JavaScript/jQuery for dynamic UI behavior (show/hide, add/remove, update attributes/elements).
- Use semantic HTML consistently across the website.
- Ensure all required media and advanced components (tables, iframes, image maps, icons, multimedia) are represented in the final system.

Traceability: `NFR-5`, `NFR-6`, `NFR-7`, `NFR-8`, `NFR-11`, `NFR-12`, `NFR-15`
